📋 Heading Elements

1. <h1> to <h6> - Headings

Syntax:

<h1>Main Heading</h1>
<h2>Section Heading</h2>
<h3>Subsection Heading</h3>
<h4>Sub-subsection Heading</h4>
<h5>Minor Heading</h5>
<h6>Smallest Heading</h6>

Output:

Main Heading (h1)

Section Heading (h2)

Subsection Heading (h3)

Sub-subsection Heading (h4)

Minor Heading (h5)
Smallest Heading (h6)

Notes:

Use hierarchically (h1 → h2 → h3). Only one h1 per page for SEO. Screen readers use these for navigation.

📄 Paragraph and Line Elements

2. <p> - Paragraph

Syntax:

<p>This is a paragraph of text.</p>

Output:

This is a paragraph of text.

Notes:

Block-level element. Browsers add margin above and below paragraphs automatically.

3. <br> - Line Break

Syntax:

First line<br>
Second line<br>
Third line

Output:

First line
Second line
Third line

Notes:

Self-closing tag. Use sparingly - prefer paragraphs for text blocks.

4. <hr> - Horizontal Rule

Syntax:

<hr>

Output:

Notes:

Self-closing tag. Represents thematic break or section divider.

5. <wbr> - Word Break Opportunity

Syntax:

supercalifragilisticexpiali<wbr>docious

Output:

supercalifragilisticexpialidocious

Notes:

Suggests where browser can break long words if needed. Invisible unless line wrapping occurs.

💪 Emphasis and Importance

6. <strong> - Strong Importance

Syntax:

<strong>This is very important!</strong>

Output:

This is very important!

Notes:

Semantic emphasis. Screen readers announce with strong emphasis.

7. <em> - Emphasis

Syntax:

<em>This is emphasized text</em>

Output:

This is emphasized text

Notes:

Semantic emphasis. Different from <i> - this has meaning, <i> is just styling.

8. <b> - Bold Text

Syntax:

<b>This is bold text</b>

Output:

This is bold text

Notes:

Stylistic bold without semantic meaning. Use <strong> for importance.

9. <i> - Italic Text

Syntax:

<i>This is italic text</i>

Output:

This is italic text

Notes:

Stylistic italic without semantic meaning. Use <em> for emphasis.

10. <u> - Underlined Text

Syntax:

<u>This is underlined text</u>

Output:

This is underlined text

Notes:

Avoid using - users expect underlined text to be links.

🔤 Text Modifications

11. <mark> - Highlighted Text

Syntax:

<mark>highlighted text</mark>

Output:

This is highlighted text in a sentence.

Notes:

HTML5 element. Represents text marked for reference or notation.

12. <small> - Small Text

Syntax:

<small>This is small text</small>

Output:

This is small text

Notes:

Represents side comments, legal text, or fine print.

13. <big> - Big Text (Deprecated)

Syntax:

<big>This is big text</big>

Output:

This is big text

Notes:

Deprecated in HTML5. Use CSS font-size instead.

14. <del> - Deleted Text

Syntax:

<del>This text was deleted</del>

Output:

This text was deleted

Notes:

Represents removed content. Can include datetime and cite attributes.

15. <ins> - Inserted Text

Syntax:

<ins>This text was inserted</ins>

Output:

This text was inserted

Notes:

Represents added content. Often used with <del> to show changes.

🔢 Subscript and Superscript

16. <sub> - Subscript

Syntax:

H<sub>2</sub>O

Output:

H2O (water molecule)

Notes:

Perfect for chemical formulas and mathematical expressions.

17. <sup> - Superscript

Syntax:

E = mc<sup>2</sup>

Output:

E = mc2 (Einstein's equation)

Notes:

Great for exponents, ordinal numbers (1st, 2nd), and footnotes.

📖 Quotations and Citations

18. <q> - Inline Quote

Syntax:

<q>To be or not to be</q>

Output:

Shakespeare wrote To be or not to be in Hamlet.

Notes:

Browsers automatically add quotation marks. Use cite attribute for source URL.

19. <blockquote> - Block Quote

Syntax:

<blockquote cite="https://example.com">
    This is a longer quotation that stands alone.
</blockquote>

Output:

This is a longer quotation that stands alone as its own paragraph or section.

Notes:

Block-level element for longer quotes. Use cite attribute for source URL.

20. <cite> - Citation

Syntax:

<cite>The Great Gatsby</cite>

Output:

My favorite book is The Great Gatsby by F. Scott Fitzgerald.

Notes:

For titles of works (books, movies, songs). Not for author names.

💻 Code and Technical Text

21. <code> - Inline Code

Syntax:

<code>console.log("Hello");</code>

Output:

Use console.log("Hello"); to print to console.

Notes:

For short code snippets within text. Use monospace font.

22. <pre> - Preformatted Text

Syntax:

<pre>
function hello() {
    console.log("Hello World!");
}
</pre>

Output:

function hello() {
    console.log("Hello World!");
}

Notes:

Preserves whitespace and line breaks. Perfect for code blocks and ASCII art.

23. <kbd> - Keyboard Input

Syntax:

<kbd>Ctrl+C</kbd>

Output:

Press Ctrl+C to copy text.

Notes:

Represents user input from keyboard, voice, or other input device.

24. <samp> - Sample Output

Syntax:

<samp>Hello World!</samp>

Output:

The program outputs: Hello World!

Notes:

Represents sample output from programs or computing systems.

25. <var> - Variable

Syntax:

<var>x</var> = 5

Output:

In the equation x + y = 10, if x = 5, then y = 5.

Notes:

Represents variables in mathematical expressions or programming contexts.

📅 Time and Abbreviations

26. <time> - Time Element

Syntax:

<time datetime="2024-01-15">January 15, 2024</time>

Output:

The event is on January 15, 2024.

Notes:

Machine-readable time format in datetime attribute. Helps search engines and assistive technology.

27. <abbr> - Abbreviation

Syntax:

<abbr title="HyperText Markup Language">HTML</abbr>

Output:

HTML is the standard markup language.

Notes:

Hover over the abbreviation to see the full form. Title attribute provides expansion.

📍 Address Information

28. <address> - Contact Information

Syntax:

<address>
    123 Main Street<br>
    City, State 12345<br>
    <a href="mailto:contact@example.com">contact@example.com</a>
</address>

Output:

123 Main Street
City, State 12345
contact@example.com

Notes:

For contact information of the document or article author. Not for postal addresses in general.

💡 Text Formatting Best Practices

  • Use semantic tags (<strong>, <em>) over presentational ones (<b>, <i>)
  • Maintain proper heading hierarchy (h1 → h2 → h3)
  • Use <code> for inline code, <pre> for code blocks
  • Provide meaningful abbreviation expansions with title attribute
  • Use <time> with datetime for machine-readable dates
