üéØ Introduction to Semantic HTML

Semantic HTML elements provide meaning to the structure of web content, making it more accessible to screen readers, search engines, and other tools. These HTML5 elements replace generic <div> elements with meaningful alternatives.

üìã Document Structure Elements

1. <header> - Page/Section Header

Syntax:

<header>
    <h1>Website Title</h1>
    <p>Website tagline or description</p>
</header>

Output:

Example Header

This is what a header section looks like

Notes:

Contains introductory content for page or section. Can include headings, logos, navigation, search forms. Multiple headers allowed per page.

2. <nav> - Navigation Links

Syntax:

<nav>
    <ul>
        <li><a href="home.html">Home</a></li>
        <li><a href="about.html">About</a></li>
        <li><a href="contact.html">Contact</a></li>
    </ul>
</nav>

Output:

Example Navigation:

  • Home
  • About
  • Contact

Notes:

For major navigation blocks. Not every link needs to be in <nav> - use for primary navigation, breadcrumbs, table of contents.

3. <main> - Main Content

Syntax:

<main>
    <h1>Page Title</h1>
    <p>Main content of the page goes here.</p>
</main>

Output:

Example Main Content

This represents the primary content area of the page.

Notes:

Only one <main> per page. Contains the dominant content. Excludes headers, footers, navigation, sidebars that are repeated across pages.

4. <section> - Thematic Section

Syntax:

<section>
    <h2>Section Title</h2>
    <p>Content related to this section.</p>
</section>

Output:

Example Section

This is a thematic grouping of content with its own heading.

Notes:

Groups related content with a theme. Should typically have a heading. Use when no other semantic element fits.

5. <article> - Independent Content

Syntax:

<article>
    <h2>Article Title</h2>
    <p>Self-contained content that could stand alone.</p>
</article>

Output:

Example Article

This is independent, self-contained content like a blog post or news article.

Notes:

For standalone content: blog posts, news articles, forum posts, comments. Should make sense when distributed independently.

6. <aside> - Sidebar Content

Syntax:

<aside>
    <h3>Related Links</h3>
    <ul>
        <li><a href="#">Related Article 1</a></li>
        <li><a href="#">Related Article 2</a></li>
    </ul>
</aside>

Output:

Example Aside

This is supplementary content like a sidebar, pull quote, or advertisement.

Notes:

For content tangentially related to main content. Sidebars, pull quotes, advertisements, author bios.

7. <footer> - Page/Section Footer

Syntax:

<footer>
    <p>&copy; 2024 Company Name. All rights reserved.</p>
    <nav>
        <a href="privacy.html">Privacy</a> |
        <a href="terms.html">Terms</a>
    </nav>
</footer>

Output:

© 2024 Example Company. All rights reserved.

Privacy | Terms

Notes:

Contains footer information for page or section. Copyright, contact info, related links, author info.

üîç Interactive Elements

8. <details> - Disclosure Widget

Syntax:

<details>
    <summary>Click to expand</summary>
    <p>This content is hidden by default and shown when expanded.</p>
</details>

Output:

Click to expand

This content is hidden by default and shown when expanded. Perfect for FAQs, collapsible sections, or progressive disclosure.

Notes:

HTML5 interactive element. No JavaScript required. Great for FAQs, collapsible content, progressive disclosure.

9. <summary> - Details Summary

Syntax:

<summary>Summary text that's always visible</summary>

Output:

See the clickable text in the <details> example above.

Notes:

Must be first child of <details>. Provides the clickable summary. If omitted, browser provides default text.

10. <dialog> - Dialog Box

Syntax:

<dialog id="myDialog">
    <p>This is a dialog box.</p>
    <button onclick="document.getElementById('myDialog').close()">Close</button>
</dialog>
<button onclick="document.getElementById('myDialog').showModal()">Open Dialog</button>

Output:

This is a dialog box example.

Notes:

HTML5 element for modal dialogs. Use showModal() for modal, show() for non-modal. Requires JavaScript to control.

üìä Complete Semantic Structure Example

Full Page Structure

Syntax:

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>Semantic Page</title>
</head>
<body>
    <header>
        <h1>Site Title</h1>
        <nav>
            <ul>
                <li><a href="home.html">Home</a></li>
                <li><a href="about.html">About</a></li>
            </ul>
        </nav>
    </header>
    
    <main>
        <article>
            <header>
                <h2>Article Title</h2>
                <p>Published on <time datetime="2024-01-15">January 15, 2024</time></p>
            </header>
            
            <section>
                <h3>Introduction</h3>
                <p>Article introduction...</p>
            </section>
            
            <section>
                <h3>Main Content</h3>
                <p>Article main content...</p>
            </section>
            
            <footer>
                <p>Author: John Doe</p>
            </footer>
        </article>
    </main>
    
    <aside>
        <h3>Related Articles</h3>
        <ul>
            <li><a href="article1.html">Related Article 1</a></li>
            <li><a href="article2.html">Related Article 2</a></li>
        </ul>
    </aside>
    
    <footer>
        <p>&copy; 2024 Website Name. All rights reserved.</p>
    </footer>
</body>
</html>

Output:

Site Title

  • Home
  • About
Article Title

Published on January 15, 2024

Introduction

Article introduction content...

Main Content

Article main content...

Author: John Doe

Related Articles
  • Related Article 1
  • Related Article 2

© 2024 Website Name. All rights reserved.

Notes:

This shows proper nesting and usage of semantic elements. Notice how headers and footers can be used at different levels.

üîÑ Semantic vs Non-Semantic Comparison

Before HTML5 (Non-Semantic)

Syntax:

<div id="header">
    <h1>Site Title</h1>
    <div id="nav">...</div>
</div>
<div id="main">
    <div class="article">...</div>
</div>
<div id="sidebar">...</div>
<div id="footer">...</div>

Notes:

Generic <div> elements with IDs/classes. No semantic meaning for assistive technologies.

HTML5 (Semantic)

Syntax:

<header>
    <h1>Site Title</h1>
    <nav>...</nav>
</header>
<main>
    <article>...</article>
</main>
<aside>...</aside>
<footer>...</footer>

Notes:

Meaningful elements that convey structure and purpose. Better for SEO, accessibility, and maintainability.

üéØ When to Use Each Element

Element Selection Guide

<header>
Page banner, article header, section introduction
<nav>
Primary navigation, breadcrumbs, table of contents
<main>
Primary content area (only one per page)
<section>
Thematic content grouping with heading
<article>
Independent, distributable content
<aside>
Supplementary content, sidebars, related info
<footer>
Page/section footer, copyright, contact info
<details>
Collapsible content, FAQs, progressive disclosure
<dialog>
Modal dialogs, popup windows, alerts

üí° Semantic HTML Best Practices

  • Use semantic elements instead of generic <div> when possible
  • Only one <main> element per page
  • Headers and footers can be used at multiple levels
  • Choose <article> for standalone content, <section> for thematic grouping
  • Use <nav> for major navigation blocks, not every link
  • Semantic elements improve accessibility and SEO
  • Test with screen readers to verify semantic structure
