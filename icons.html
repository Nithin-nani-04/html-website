üîñ Favicons

1. Basic Favicon

Syntax:

<link rel="icon" type="image/x-icon" href="/favicon.ico">
<link rel="shortcut icon" href="/favicon.ico">

Output:

Check the browser tab - you should see a blue "H" icon!

Notes:

Traditional .ico format. Place in root directory. Shows in browser tabs, bookmarks, and history.

2. Modern Favicon with Multiple Sizes

Syntax:

<link rel="icon" type="image/png" sizes="32x32" href="/favicon-32x32.png">
<link rel="icon" type="image/png" sizes="16x16" href="/favicon-16x16.png">
<link rel="apple-touch-icon" sizes="180x180" href="/apple-touch-icon.png">
<link rel="manifest" href="/site.webmanifest">

Output:

Multiple sizes ensure crisp display on all devices and contexts.

Notes:

PNG format with multiple sizes. Apple touch icon for iOS. Web manifest for PWA support.

3. SVG Favicon

Syntax:

<link rel="icon" type="image/svg+xml" href="/favicon.svg">

Output:

Scalable vector favicon that adapts to any size perfectly.

Notes:

Modern browsers support SVG favicons. Scalable and can adapt to dark/light themes.

üéØ Font Awesome Icons

4. Font Awesome Setup

Syntax:

<link rel="stylesheet" href="https://cdnjs.cloudflare.com/ajax/libs/font-awesome/6.4.0/css/all.min.css">

Output:

Include this in your <head> to use Font Awesome icons.

Notes:

Popular icon library with 1000+ icons. Free and Pro versions available.

5. Basic Font Awesome Icons

Syntax:

<i class="fas fa-home"></i> Home
<i class="fas fa-user"></i> User
<i class="fas fa-envelope"></i> Email
<i class="fas fa-phone"></i> Phone

Output:

Home

User

Email

Phone

Notes:

fas = Font Awesome Solid. Also available: far (regular), fab (brands), fal (light - Pro only).

<i class="fas fa-star fa-lg"></i> Large
<i class="fas fa-star fa-2x"></i> 2x

Notes:

Size classes: fa-xs, fa-sm, fa-lg, fa-2x through fa-10x for scaling.

üé® Material Icons

9. Material Icons Setup

Syntax:

<link rel="stylesheet" href="https://fonts.googleapis.com/icon?family=Material+Icons">

Output:

Include this in your <head> to use Google Material Icons.

Notes:

Google's Material Design icon set. Clean, modern icons following Material Design principles.

10. Basic Material Icons

Syntax:

<span class="material-icons">home</span> Home
<span class="material-icons">person</span> Person
<span class="material-icons">email</span> Email
<span class="material-icons">phone</span> Phone
<span class="material-icons">favorite</span> Favorite

Output:

home Home

person Person

email Email

phone Phone

favorite Favorite

Notes:

Use icon names as text content. Icons are font-based and can be styled with CSS.

11. Material Icons Variants

Syntax:

<span class="material-icons">star</span> Filled
<span class="material-icons-outlined">star</span> Outlined
<span class="material-icons-round">star</span> Round
<span class="material-icons-sharp">star</span> Sharp
<span class="material-icons-two-tone">star</span> Two Tone

Output:

star Filled

star Outlined

star Round

star Sharp

star Two Tone

Notes:

Five style variants available. Each requires separate font import for optimal loading.

    <path d="M12 2l3.09 6.26L22 9.27l-5 4.87 1.18 6.88L12 17.77l-6.18 3.25L7 14.14 2 9.27l6.91-1.01L12 2z"/>

13. SVG Icon with CSS Styling

Syntax:

<svg class="icon icon-heart" viewBox="0 0 24 24">
    <path d="M12 21.35l-1.45-1.32C5.4 15.36 2 12.28 2 8.5 2 5.42 4.42 3 7.5 3c1.74 0 3.41.81 4.5 2.09C13.09 3.81 14.76 3 16.5 3 19.58 3 22 5.42 22 8.5c0 3.78-3.4 6.86-8.55 11.54L12 21.35z"/>
</svg>

<style>
.icon-heart {
    width: 24px;
    height: 24px;
    fill: #e74c3c;
    transition: transform 0.2s;
}
.icon-heart:hover {
    transform: scale(1.2);
    fill: #c0392b;
}
</style>

Output:

Hover over the heart!

Notes:

SVG icons can be styled with CSS. Use fill for color, transform for animations.

üí° Icon Best Practices

  • Use icons consistently throughout your site
  • Ensure icons are accessible with proper alt text or aria-labels
  • Choose icon libraries that match your design style
  • Optimize icon loading for better performance
  • Use SVG icons for scalability and customization
  • Test icons across different devices and screen sizes
  • Consider icon meaning across different cultures
  • Provide text labels alongside icons when possible
