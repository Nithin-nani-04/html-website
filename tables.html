📋 Basic Table Elements

1. <table> - Table Container

Syntax:

<table>
    <tr>
        <td>Cell 1</td>
        <td>Cell 2</td>
    </tr>
</table>

Output:

Cell 1 Cell 2

Notes:

Container for all table content. Use only for tabular data, not for layout. Border attribute is deprecated - use CSS instead.

2. <tr> - Table Row

Syntax:

<tr>
    <td>Row 1, Cell 1</td>
    <td>Row 1, Cell 2</td>
</tr>

Output:

Row 1, Cell 1 Row 1, Cell 2

Notes:

Defines a table row. Contains table cells (<td> or <th>). Must be direct child of <table>, <thead>, <tbody>, or <tfoot>.

3. <td> - Table Data Cell

Syntax:

<td>Cell content</td>

Output:

Regular data cell Another data cell

Notes:

Contains table data. Can contain any HTML content. Use for regular table cells.

4. <th> - Table Header Cell

Syntax:

<tr>
    <th>Header 1</th>
    <th>Header 2</th>
</tr>

Output:

Header 1 Header 2
Data 1 Data 2

Notes:

Header cells are bold and centered by default. Important for accessibility - screen readers use these to understand table structure.

📝 Table Caption and Structure

5. <caption> - Table Caption

Syntax:

<table>
    <caption>Monthly Sales Report</caption>
    <tr>
        <th>Month</th>
        <th>Sales</th>
    </tr>
    <tr>
        <td>January</td>
        <td>$10,000</td>
    </tr>
</table>

Output:

Monthly Sales Report
Month Sales
January $10,000

Notes:

Must be first child of <table>. Provides title/description for the table. Important for accessibility.

🏗️ Table Grouping Elements

6. <thead> - Table Header Group

Syntax:

<table>
    <thead>
        <tr>
            <th>Product</th>
            <th>Price</th>
            <th>Stock</th>
        </tr>
    </thead>
    <tbody>
        <tr>
            <td>Laptop</td>
            <td>$999</td>
            <td>15</td>
        </tr>
    </tbody>
</table>

Output:

Product Price Stock
Laptop $999 15
Mouse $25 50

Notes:

Groups header content. Helps with styling and accessibility. Can be styled separately from body content.

7. <tbody> - Table Body Group

Syntax:

<tbody>
    <tr>
        <td>Data row 1</td>
        <td>Data row 1</td>
    </tr>
    <tr>
        <td>Data row 2</td>
        <td>Data row 2</td>
    </tr>
</tbody>

Output:

See the data rows in the table above.

Notes:

Groups main table content. Can have multiple <tbody> elements for different sections of data.

8. <tfoot> - Table Footer Group

Syntax:

<table>
    <thead>
        <tr>
            <th>Item</th>
            <th>Quantity</th>
            <th>Price</th>
        </tr>
    </thead>
    <tbody>
        <tr>
            <td>Apples</td>
            <td>5</td>
            <td>$2.50</td>
        </tr>
        <tr>
            <td>Bananas</td>
            <td>3</td>
            <td>$1.50</td>
        </tr>
    </tbody>
    <tfoot>
        <tr>
            <th>Total</th>
            <th>8</th>
            <th>$4.00</th>
        </tr>
    </tfoot>
</table>

Output:

Item Quantity Price
Apples 5 $2.50
Bananas 3 $1.50
Total 8 $4.00

Notes:

Groups footer content like totals or summaries. Appears at bottom of table regardless of position in HTML.

📐 Column Elements

9. <colgroup> - Column Group

Syntax:

<table>
    <colgroup>
        <col style="background-color: #f0f0f0;">
        <col span="2" style="background-color: #e0e0e0;">
    </colgroup>
    <tr>
        <th>Name</th>
        <th>Age</th>
        <th>City</th>
    </tr>
    <tr>
        <td>John</td>
        <td>25</td>
        <td>New York</td>
    </tr>
</table>

Output:

Name Age City
John 25 New York

Notes:

Groups columns for styling. Must appear after <caption> and before table content. Contains <col> elements.

10. <col> - Column

Syntax:

<col span="2" style="background-color: yellow;">

Output:

See the styled columns in the <colgroup> example above.

Notes:

Self-closing tag. Defines properties for one or more columns. span attribute specifies number of columns.

🔗 Cell Spanning and Attributes

Colspan - Spanning Columns

Syntax:

<table>
    <tr>
        <th colspan="3">Quarterly Report</th>
    </tr>
    <tr>
        <th>Q1</th>
        <th>Q2</th>
        <th>Q3</th>
    </tr>
    <tr>
        <td>$100K</td>
        <td>$120K</td>
        <td>$110K</td>
    </tr>
</table>

Output:

Quarterly Report
Q1 Q2 Q3
$100K $120K $110K

Notes:

colspan makes a cell span multiple columns. Useful for headers that cover multiple data columns.

Rowspan - Spanning Rows

Syntax:

<table>
    <tr>
        <th>Department</th>
        <th>Employee</th>
        <th>Salary</th>
    </tr>
    <tr>
        <td rowspan="2">Engineering</td>
        <td>Alice</td>
        <td>$80K</td>
    </tr>
    <tr>
        <td>Bob</td>
        <td>$75K</td>
    </tr>
    <tr>
        <td rowspan="2">Marketing</td>
        <td>Carol</td>
        <td>$65K</td>
    </tr>
    <tr>
        <td>Dave</td>
        <td>$60K</td>
    </tr>
</table>

Output:

Department Employee Salary
Engineering Alice $80K
Bob $75K
Marketing Carol $65K
Dave $60K

Notes:

rowspan makes a cell span multiple rows. Useful for grouping related data vertically.

♿ Accessibility Attributes

Headers Attribute

Syntax:

<table>
    <tr>
        <th id="name">Name</th>
        <th id="age">Age</th>
        <th id="city">City</th>
    </tr>
    <tr>
        <td headers="name">John Doe</td>
        <td headers="age">30</td>
        <td headers="city">Boston</td>
    </tr>
</table>

Output:

Name Age City
John Doe 30 Boston

Notes:

headers attribute explicitly associates data cells with header cells. Important for complex tables and screen readers.

Scope Attribute

Syntax:

<table>
    <tr>
        <th scope="col">Product</th>
        <th scope="col">Q1</th>
        <th scope="col">Q2</th>
    </tr>
    <tr>
        <th scope="row">Laptops</th>
        <td>100</td>
        <td>120</td>
    </tr>
    <tr>
        <th scope="row">Phones</th>
        <td>200</td>
        <td>180</td>
    </tr>
</table>

Output:

Product Q1 Q2
Laptops 100 120
Phones 200 180

Notes:

scope attribute specifies whether header applies to row, column, rowgroup, or colgroup. Values: row, col, rowgroup, colgroup.

📊 Complex Table Example

Complete Table with All Features

Syntax:

<table>
    <caption>Company Sales Data 2024</caption>
    <colgroup>
        <col style="background-color: #f9f9f9;">
        <col span="4" style="background-color: #fff;">
    </colgroup>
    <thead>
        <tr>
            <th scope="col">Region</th>
            <th scope="col">Q1</th>
            <th scope="col">Q2</th>
            <th scope="col">Q3</th>
            <th scope="col">Q4</th>
        </tr>
    </thead>
    <tbody>
        <tr>
            <th scope="row">North</th>
            <td>$150K</td>
            <td>$180K</td>
            <td>$200K</td>
            <td>$220K</td>
        </tr>
        <tr>
            <th scope="row">South</th>
            <td>$120K</td>
            <td>$140K</td>
            <td>$160K</td>
            <td>$180K</td>
        </tr>
        <tr>
            <th scope="row">East</th>
            <td>$100K</td>
            <td>$110K</td>
            <td>$130K</td>
            <td>$140K</td>
        </tr>
        <tr>
            <th scope="row">West</th>
            <td>$200K</td>
            <td>$250K</td>
            <td>$280K</td>
            <td>$300K</td>
        </tr>
    </tbody>
    <tfoot>
        <tr>
            <th scope="row">Total</th>
            <td><strong>$570K</strong></td>
            <td><strong>$680K</strong></td>
            <td><strong>$770K</strong></td>
            <td><strong>$840K</strong></td>
        </tr>
    </tfoot>
</table>

Output:

Company Sales Data 2024
Region Q1 Q2 Q3 Q4
North $150K $180K $200K $220K
South $120K $140K $160K $180K
East $100K $110K $130K $140K
West $200K $250K $280K $300K
Total $570K $680K $770K $840K

Notes:

This example demonstrates proper table structure with caption, column grouping, semantic sections, and accessibility attributes.

💡 Table Best Practices

  • Use tables only for tabular data, not for layout
  • Always include table headers (<th>) for accessibility
  • Use <caption> to describe the table's purpose
  • Group related content with <thead>, <tbody>, <tfoot>
  • Use scope attribute on headers for better accessibility
  • Consider responsive design for mobile devices
  • Use CSS for styling instead of deprecated HTML attributes
  • Test with screen readers to ensure accessibility
