üìã Form Container

1. <form> - Form Container

Syntax:

<form action="/submit" method="post" enctype="multipart/form-data">
    <!-- Form elements go here -->
</form>

Output:

This is a form container (other examples will show actual form elements)

Notes:

Container for all form elements. action: where to send data. method: GET or POST. enctype: encoding type for file uploads.

üè∑Ô∏è Labels and Basic Structure

2. <label> - Input Label

Syntax:

<label for="username">Username:</label>
<input type="text" id="username" name="username">

<!-- Alternative: wrapping label -->
<label>
    Email:
    <input type="email" name="email">
</label>

Output:

Notes:

Associates text with form controls. Clicking label focuses the input. Use for attribute or wrap the input.

üìù Text Input Types

3. <input type="text"> - Text Input

Syntax:

<input type="text" name="firstname" placeholder="Enter your first name" maxlength="50">

Output:

Notes:

Default input type. Single-line text. Use placeholder for hints, maxlength to limit characters.

4. <input type="password"> - Password Input

Syntax:

<input type="password" name="password" placeholder="Enter password" minlength="8">

Output:

Notes:

Masks input characters. Use minlength for minimum password length requirements.

5. <input type="email"> - Email Input

Syntax:

<input type="email" name="email" placeholder="user@example.com" required>

Output:

Notes:

HTML5 input with email validation. Browsers check for valid email format. Mobile keyboards show @ symbol.

6. <input type="url"> - URL Input

Syntax:

<input type="url" name="website" placeholder="https://example.com">

Output:

Notes:

HTML5 input for URLs. Validates URL format. Mobile keyboards show .com key.

7. <input type="tel"> - Telephone Input

Syntax:

<input type="tel" name="phone" placeholder="+1 (555) 123-4567" pattern="[0-9]{3}-[0-9]{3}-[0-9]{4}">

Output:

Notes:

For phone numbers. Mobile devices show numeric keypad. Use pattern attribute for format validation.

8. <input type="search"> - Search Input

Syntax:

<input type="search" name="query" placeholder="Search..." autocomplete="off">

Output:

Notes:

Semantic search input. May show clear button (X) when text is entered. Rounded corners on some browsers.

üî¢ Numeric Input Types

9. <input type="number"> - Number Input

Syntax:

<input type="number" name="age" min="0" max="120" step="1" value="25">

Output:

Notes:

Numeric input with spinner controls. min/max set range, step sets increment, value sets default.

10. <input type="range"> - Range Slider

Syntax:

<input type="range" name="volume" min="0" max="100" value="50" step="10">

Output:

50

Notes:

Slider control for selecting values within a range. Great for settings like volume, brightness, etc.

üìÖ Date and Time Inputs

11. <input type="date"> - Date Input

Syntax:

<input type="date" name="birthday" min="1900-01-01" max="2024-12-31">

Output:

Notes:

HTML5 date picker. Format: YYYY-MM-DD. Browsers provide calendar widget.

12. <input type="time"> - Time Input

Syntax:

<input type="time" name="appointment" min="09:00" max="17:00" step="900">

Output:

Notes:

Time picker in 24-hour format. step in seconds (900 = 15 minutes). Format: HH:MM or HH:MM:SS.

13. <input type="datetime-local"> - Local DateTime

Syntax:

<input type="datetime-local" name="meeting" value="2024-01-15T14:30">

Output:

Notes:

Date and time without timezone. Format: YYYY-MM-DDTHH:MM. Combines date and time pickers.

14. <input type="month"> - Month Input

Syntax:

<input type="month" name="start-month" value="2024-01">

Output:

Notes:

Month and year picker. Format: YYYY-MM. Useful for credit card expiry, reports, etc.

15. <input type="week"> - Week Input

Syntax:

<input type="week" name="project-week" value="2024-W03">

Output:

Notes:

Week picker. Format: YYYY-WNN (W03 = 3rd week). Useful for scheduling and planning.

üé® Visual Input Types

16. <input type="color"> - Color Picker

Syntax:

<input type="color" name="theme-color" value="#ff0000">

Output:

Notes:

Color picker widget. Returns hex color value (#RRGGBB). Browsers provide color palette interface.

17. <input type="file"> - File Upload

Syntax:

<input type="file" name="upload" accept=".jpg,.png,.pdf" multiple>

Output:

Notes:

File selection. accept limits file types, multiple allows multiple files. Form needs enctype="multipart/form-data".

‚òëÔ∏è Choice Input Types

18. <input type="checkbox"> - Checkbox

Syntax:

<input type="checkbox" id="newsletter" name="newsletter" value="yes" checked>
<label for="newsletter">Subscribe to newsletter</label>

<input type="checkbox" id="terms" name="terms" value="accepted" required>
<label for="terms">I agree to the terms</label>

Output:

Notes:

Boolean choice. checked attribute sets default state. Multiple checkboxes can be selected.

19. <input type="radio"> - Radio Button

Syntax:

<input type="radio" id="small" name="size" value="small">
<label for="small">Small</label>

<input type="radio" id="medium" name="size" value="medium" checked>
<label for="medium">Medium</label>

<input type="radio" id="large" name="size" value="large">
<label for="large">Large</label>

Output:

Notes:

Mutually exclusive choices. Same name groups radio buttons. Only one can be selected per group.

üîò Button Input Types

20. <input type="submit"> - Submit Button

Syntax:

<input type="submit" value="Submit Form">

Output:

Notes:

Submits the form. value attribute sets button text. Default behavior when Enter is pressed in form.

21. <input type="reset"> - Reset Button

Syntax:

<input type="reset" value="Clear Form">

Output:

Notes:

Resets all form fields to default values. Use sparingly - can be frustrating for users.

22. <input type="button"> - Generic Button

Syntax:

<input type="button" value="Click Me" onclick="alert('Button clicked!')">

Output:

Notes:

Generic button with no default behavior. Requires JavaScript for functionality. Consider <button> element instead.

23. <input type="image"> - Image Button

Syntax:

<input type="image" src="submit-button.png" alt="Submit" width="100" height="30">

Output:

Notes:

Image that acts as submit button. Sends x,y coordinates of click. alt attribute required for accessibility.

üîí Hidden Input

24. <input type="hidden"> - Hidden Field

Syntax:

<input type="hidden" name="csrf_token" value="abc123xyz">
<input type="hidden" name="user_id" value="12345">

Output:

Hidden fields are not visible (but present in form data)

Notes:

Not visible to users but included in form submission. Used for tokens, IDs, and other data.

üìù Other Form Elements

25. <textarea> - Multi-line Text

Syntax:

<textarea name="message" rows="4" cols="50" placeholder="Enter your message..." maxlength="500">Default text</textarea>

Output:

Notes:

Multi-line text input. rows/cols set size, but CSS is preferred. Content goes between opening and closing tags.

26. <select> - Dropdown List

Syntax:

<select name="country" required>
    <option value="">Choose a country</option>
    <option value="us">United States</option>
    <option value="ca" selected>Canada</option>
    <option value="uk">United Kingdom</option>
</select>

Output:

Notes:

Dropdown selection. selected attribute sets default. Empty value option useful for validation.

27. <option> - Select Option

Syntax:

<option value="value1" selected disabled>Option text</option>

Output:

See usage in <select> example above.

Notes:

Individual options in select. value sent to server, text shown to user. selected/disabled attributes available.

28. <optgroup> - Option Group

Syntax:

<select name="food">
    <optgroup label="Fruits">
        <option value="apple">Apple</option>
        <option value="banana">Banana</option>
    </optgroup>
    <optgroup label="Vegetables">
        <option value="carrot">Carrot</option>
        <option value="broccoli">Broccoli</option>
    </optgroup>
</select>

Output:

Notes:

Groups related options in select. label attribute provides group heading. Cannot be selected.

29. <datalist> - Data List

Syntax:

<input type="text" name="browser" list="browsers" placeholder="Choose or type browser">
<datalist id="browsers">
    <option value="Chrome">
    <option value="Firefox">
    <option value="Safari">
    <option value="Edge">
</datalist>

Output:

Notes:

Provides autocomplete suggestions for input. Users can type freely or choose from list. HTML5 element.

30. <button> - Button Element

Syntax:

<button type="submit">Submit Form</button>
<button type="reset">Reset</button>
<button type="button" onclick="alert('Clicked!')">Click Me</button>

Output:

Notes:

More flexible than input buttons. Can contain HTML content. type: submit (default), reset, or button.

üìä Form Organization

31. <fieldset> - Field Group

Syntax:

<fieldset>
    <legend>Personal Information</legend>
    <label for="fname">First Name:</label>
    <input type="text" id="fname" name="fname">
    
    <label for="lname">Last Name:</label>
    <input type="text" id="lname" name="lname">
</fieldset>

Output:

Personal Information

Notes:

Groups related form elements. Creates visual border. Important for accessibility and screen readers.

32. <legend> - Fieldset Caption

Syntax:

<legend>Contact Information</legend>

Output:

See the "Personal Information" caption in the fieldset example above.

Notes:

Must be first child of <fieldset>. Provides caption for the grouped fields. Important for accessibility.

üìà Progress and Output

33. <progress> - Progress Bar

Syntax:

<progress value="70" max="100">70%</progress>
<progress>Loading...</progress> <!-- Indeterminate -->

Output:

70% 70%

Loading...

Notes:

Shows completion progress. value/max for determinate, no value for indeterminate (loading animation).

34. <meter> - Gauge/Meter

Syntax:

<meter value="6" min="0" max="10">6 out of 10</meter>
<meter value="0.6">60%</meter>

Output:

6 out of 10 6/10

60% 60%

Notes:

For scalar measurements within known range. Not for progress - use <progress> instead.

35. <output> - Output Element

Syntax:

<form oninput="result.value=parseInt(a.value)+parseInt(b.value)">
    <input type="range" id="a" value="50"> +
    <input type="number" id="b" value="25"> =
    <output name="result" for="a b">75</output>
</form>

Output:

+ = 75

Notes:

Displays calculation results. for attribute lists related input IDs. Updated via JavaScript.

Registration Form with All Elements

Syntax:

<form action="/register" method="post" enctype="multipart/form-data">
    <fieldset>
        <legend>Account Information</legend>
        
        <label for="username">Username:</label>
        <input type="text" id="username" name="username" required minlength="3">
        
        <label for="email">Email:</label>
        <input type="email" id="email" name="email" required>
        
        <label for="password">Password:</label>
        <input type="password" id="password" name="password" required minlength="8">
    </fieldset>
    
    <fieldset>
        <legend>Personal Details</legend>
        
        <label for="birthdate">Birth Date:</label>
        <input type="date" id="birthdate" name="birthdate">
        
        <label for="country">Country:</label>
        <select id="country" name="country">
            <option value="">Select Country</option>
            <option value="us">United States</option>
            <option value="ca">Canada</option>
        </select>
        
        <fieldset>
            <legend>Gender</legend>
            <input type="radio" id="male" name="gender" value="male">
            <label for="male">Male</label>
            
            <input type="radio" id="female" name="gender" value="female">
            <label for="female">Female</label>
            
            <input type="radio" id="other" name="gender" value="other">
            <label for="other">Other</label>
        </fieldset>
    </fieldset>
    
    <fieldset>
        <legend>Preferences</legend>
        
        <input type="checkbox" id="newsletter" name="newsletter" value="yes">

        <textarea id="bio" name="bio" rows="4" cols="50"></textarea>
        
        <label for="avatar">Profile Picture:</label>

    </fieldset>

    <button type="reset">Clear Form</button>

Output:

Account Information




Personal Details



Gender
Preferences



Notes:

Complete form demonstrating proper structure, validation, accessibility, and all major form elements.

üí° Form Best Practices

  • Always use labels for form controls (for attribute or wrapping)
  • Group related fields with fieldset and legend
  • Use appropriate input types for better UX and validation
  • Provide clear placeholder text and validation messages
  • Use required attribute for mandatory fields
  • Test forms with keyboard navigation and screen readers
  • Consider mobile users - appropriate input types show better keyboards
  • Use autocomplete attributes to help users fill forms faster
