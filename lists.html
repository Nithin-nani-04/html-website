📝 Unordered Lists

1. <ul> - Unordered List

Syntax:

<ul>
    <li>First item</li>
    <li>Second item</li>
    <li>Third item</li>
</ul>

Output:

  • First item
  • Second item
  • Third item

Notes:

Creates bulleted lists. Order doesn't matter semantically. Can be styled with CSS list-style-type.

2. <li> - List Item

Syntax:

<li>List item content</li>

Output:

See examples above and below - each bullet point or number is a <li> element.

Notes:

Must be direct child of <ul>, <ol>, or <menu>. Can contain any content including other lists.

🔢 Ordered Lists

3. <ol> - Ordered List

Syntax:

<ol>
    <li>First step</li>
    <li>Second step</li>
    <li>Third step</li>
</ol>

Output:

  1. First step
  2. Second step
  3. Third step

Notes:

Creates numbered lists. Order matters semantically. Perfect for instructions, rankings, or sequences.

Ordered List with Custom Start

Syntax:

<ol start="5">
    <li>Fifth item</li>
    <li>Sixth item</li>
    <li>Seventh item</li>
</ol>

Output:

  1. Fifth item
  2. Sixth item
  3. Seventh item

Notes:

Use start attribute to begin numbering from a specific number.

Ordered List with Different Types

Syntax:

<ol type="A">
    <li>First item</li>
    <li>Second item</li>
</ol>

<ol type="I">
    <li>First item</li>
    <li>Second item</li>
</ol>

<ol type="a">
    <li>First item</li>
    <li>Second item</li>
</ol>

<ol type="i">
    <li>First item</li>
    <li>Second item</li>
</ol>

Output:

Uppercase letters:

  1. First item
  2. Second item

Uppercase Roman numerals:

  1. First item
  2. Second item

Lowercase letters:

  1. First item
  2. Second item

Lowercase Roman numerals:

  1. First item
  2. Second item

Notes:

Type attribute values: "1" (default), "A", "a", "I", "i". CSS list-style-type provides more options.

Reversed Ordered List

Syntax:

<ol reversed>
    <li>Third item</li>
    <li>Second item</li>
    <li>First item</li>
</ol>

Output:

  1. Third item
  2. Second item
  3. First item

Notes:

HTML5 attribute. Numbers count down instead of up. Useful for countdowns or rankings.

📖 Definition Lists

4. <dl> - Description List

Syntax:

<dl>
    <dt>HTML</dt>
    <dd>HyperText Markup Language</dd>
    
    <dt>CSS</dt>
    <dd>Cascading Style Sheets</dd>
    
    <dt>JavaScript</dt>
    <dd>Programming language for web interactivity</dd>
</dl>

Output:

HTML
HyperText Markup Language
CSS
Cascading Style Sheets
JavaScript
Programming language for web interactivity

Notes:

Perfect for glossaries, dictionaries, FAQs, or any term-definition pairs.

5. <dt> - Description Term

Syntax:

<dt>Term to be defined</dt>

Output:

See the bold terms in the definition list above.

Notes:

Must be direct child of <dl>. Can have multiple <dd> elements following it.

6. <dd> - Description Details

Syntax:

<dd>Definition or description of the term</dd>

Output:

See the indented descriptions in the definition list above.

Notes:

Must be direct child of <dl>. Multiple <dd> elements can follow one <dt>.

Complex Definition List Example

Syntax:

<dl>
    <dt>Web Browser</dt>
    <dd>Software application for accessing websites</dd>
    <dd>Examples: Chrome, Firefox, Safari, Edge</dd>
    
    <dt>HTTP</dt>
    <dt>HTTPS</dt>
    <dd>Protocols for transferring web pages</dd>
</dl>

Output:

Web Browser
Software application for accessing websites
Examples: Chrome, Firefox, Safari, Edge
HTTP
HTTPS
Protocols for transferring web pages

Notes:

Multiple <dd> elements can follow one <dt>, and multiple <dt> elements can share one <dd>.

🍔 Menu Lists

7. <menu> - Menu List

Syntax:

<menu>
    <li><button>New File</button></li>
    <li><button>Open File</button></li>
    <li><button>Save File</button></li>
</menu>

Output:

    •

    Notes:

    HTML5 element for interactive menus. Similar to <ul> but semantic meaning for toolbars and context menus.

    🔗 Nested Lists

    Nested Unordered Lists

    Syntax:

    <ul>
    <li>Fruits
        <ul>
            <li>Apples</li>
            <li>Bananas</li>
            <li>Oranges</li>
        </ul>
    </li>
    <li>Vegetables
        <ul>
            <li>Carrots</li>
            <li>Broccoli</li>
        </ul>
    </li>
</ul>

    Output:

    • Fruits
      • Apples
      • Bananas
      • Oranges
    • Vegetables
      • Carrots
      • Broccoli

    Notes:

    Nested lists must be inside <li> elements, not directly inside parent list.

    Nested Ordered Lists

    Syntax:

    <ol>
    <li>Prepare ingredients
        <ol type="a">
            <li>Wash vegetables</li>
            <li>Chop onions</li>
            <li>Measure spices</li>
        </ol>
    </li>
    <li>Cook the dish
        <ol type="a">
            <li>Heat oil in pan</li>
            <li>Add ingredients</li>
            <li>Simmer for 20 minutes</li>
        </ol>
    </li>
    <li>Serve and enjoy</li>
</ol>

    Output:

    1. Prepare ingredients
      1. Wash vegetables
      2. Chop onions
      3. Measure spices
    2. Cook the dish
      1. Heat oil in pan
      2. Add ingredients
      3. Simmer for 20 minutes
    3. Serve and enjoy

    Notes:

    Great for hierarchical instructions. Different numbering types help distinguish levels.

    Mixed List Types

    Syntax:

    <ol>
    <li>Programming Languages
        <ul>
            <li>JavaScript</li>
            <li>Python</li>
            <li>Java</li>
        </ul>
    </li>
    <li>Frameworks
        <ul>
            <li>React</li>
            <li>Vue.js</li>
            <li>Angular</li>
        </ul>
    </li>
</ol>

    Output:

    1. Programming Languages
      • JavaScript
      • Python
      • Java
    2. Frameworks
      • React
      • Vue.js
      • Angular

    Notes:

    You can mix ordered and unordered lists. Choose based on whether order matters at each level.

    💡 List Best Practices

    • Use <ul> when order doesn't matter
    • Use <ol> for sequential steps or rankings
    • Use <dl> for term-definition pairs
    • Nest lists inside <li> elements, not directly in parent lists
    • Consider semantic meaning when choosing list types
    • Use CSS for visual styling instead of deprecated attributes
